SCSS入门：

1.css预处理器：定义了一种新的编程语言，变异后成正常的css文件，为css增加一些编程的特性，无需考虑浏览器的兼容问题、让css更加简洁，适应性更强，可读性更佳，更易于代码的维护
	css预处理器语言：scss(sass)、less等
	
2.Sass和SCSS的区别：
	2.1扩展名不同：.sass 和 .scss
	2.2.sass是以严格缩进语法规则来书写的，不带大括号和分号;而SCSS的语法和css书写语法类似

3.sass的安装
	3.1.先安装ruby,因为sass依赖ruby
	3.2.安装sass在终端输入：gem install sass 命令
	3.3.本地安装sass
	
4.scss语法格式
	$变量名:变量值;如-》$color:#333;
	调用：如-》.div{color:$color;}
	
5.sass编译
	5.1.sass命令编译：在命令终端输入sass指令俩编译sass，最直接、最简单
	5.1.1.单文件编译
		sass <要编译的sass文件路径>:<要输出css文件样式>
	5.1.2.多文件编译
		sass sass/:css/ 
			sass/ --》表示项目中sass文件夹中的所有.scss或.sass文件编译成.css文件放到项目的css/文件夹中
	缺点：每次保存scss文件后都要重新编译太麻烦
	解决方法：开启“watch”功能(自动编译)：
	sass --watch <要编译的Sass文件路径>:<要输出CSS文件路径>
	tip:路径中不能带有中文
		
	
	5.2.GUI工具编译
	5.3.自动化编译
	
6.sass嵌套输出方式nested
	在编译的时候后面带上参数"--style newsted"
	
7.sass展开输出方式expanded(扩展型)
	在编译的时候后面带上参数"--style expanded"
	
8.sass展开输出方式compact(紧凑型)
	在编译的时候后面带上参数"--style compact"
	
9.sass展开输出方式compressed(压缩的)
在编译的时候后面带上参数"--style compressed"
	
10.sass变量声明：$变量名:变量值

11.普通变量和默认变量：
	普通变量声明后可以在全局范围内使用
	默认变量只需要在值后面加上 !default 即可
	默认变量一般是用来设置默认值的，然后根据需求来覆盖，覆盖的方式是在默认变量之前重新声明一下变量即可，默认变量的价值在于在进行组件化开发的时候会非常有用
	
12.局部变量和全局变量：
	局部变量：在元素内部声明的变量
	全局变量：在元素外面定义的变量
	全局变量的影子：和全局变量名字相同的变量叫做全局变量的影子。
	
13.sass嵌套
	13.1：选择器嵌套
	13.2：属性嵌套(具有相同的属性前缀)
	13.3：伪类嵌套
	
14.sass混合宏(宏表示批量处理，也就是复用)
	14.1.声明混合宏 
		@mixin 混合宏名称{ 复用样式}
		如：@mixin border-radius{....}
	14.2.调用混合宏
		@include 混合宏名称;
		如：button{@include 混合宏名称;}
	14.3.混合宏的参数
		14.3.1.不带参数，如上
		14.3.2.不带任何值的参数
			@mixin 混合宏名称(参数1，参数2){ }
		14.3.3.带默认值的参数
		@mixin 混合宏名称(参数1:值){ }

15.sass继承
	子级选择器{
		@extend .父级选择器
	}
	在sass的继承中，可以继承类样式块中的所有样式代码，而且编译出来会将css选择器合并在一起，形成组合选择器
	
16.sass占位符%
	用占位符声明的代码，如果不被@extend调用就不会被编译，类似于抽象类的继承
	通过@extend调用的占位符，编译出来的代码将相同的代码合并在一起，代码变得十分简洁

	19.2乘法，如width:10px*2
	19.3除法：如果初始中没有变量或不是在一个数学表达式中(有加减等)，就要将除式运算用小括号括起来，否则"/"不会被当做除号；tip：除法中相同单位相除不会报错，会产生一个无单位的值
	19.4变量计算
	19.5数字运算
	19.6颜色运算：所有算数运算都支持颜色值，并且是分段计算的
	19.7字符运算：用“+”对字符串进行连接，也可以直接连接字符；如果有引号的字符串和没有引号的字符串相加，看哪个在+号的左边，如果有引号的字符串在加号左边，则结果带加号；否则不加
